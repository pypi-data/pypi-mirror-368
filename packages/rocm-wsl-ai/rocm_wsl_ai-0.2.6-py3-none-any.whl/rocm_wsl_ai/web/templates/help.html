Hilfe & Tipps

Kurze Hinweise zu Logs, Ports und WSL‑Tücken. Für mehr Details schau in die README oder Issues.

Erste Schritte
  • Öffne den Wizard und setze Basisverzeichnis/venv sowie optionale Defaults (Host/Port/Flags).
  • Installiere ein Tool unter "Tools" und starte es im Hintergrund.
  • Öffne das Interface über den Link in der Karte oder nutze die Live-Logs.
Live-Logs (SSE/WebSocket)
  • Filtere Logs per Regex, aktiviere Autoscroll.
  • Falls keine Logs erscheinen: prüfen, ob das Tool gestartet ist und Logdateien existieren.
Tools & Ports
  • In den Einstellungen pro Tool kannst du URL und Extra-Args setzen (z. B. --listen --port 7860).
  • Bei Konflikten (Port belegt) wähle einen freien Port und passe die URL an.
WSL-Hinweise
  • Bind auf 0.0.0.0, wenn du vom Host zugreifen willst.
  • SillyTavern benötigt Node/NVM; Installation wird im Installer transparent ausgeführt.
Fehlerbehebung
  • Tool startet nicht: Logs prüfen, Extra-Args entfernen/testweise minimal starten.
  • Keine Verbindung: Firewall/Port prüfen, URL korrekt?
  • Installationsfehler: Job-Historie ansehen; ggf. erneut starten.
